# MOY 产品宪章（Product Charter）

> 本文件是 MOY 项目的最高纲领。所有产品决策、技术决策、架构决策、范围决策必须以本文件为准绳。冲突时以本文件为准。

---

## 1. 产品定位

**MOY（Mate Of You）** 是桐鸣科技面向 B 端企业的 **Web 端 AI 增长操作系统**。

MOY 1.0 正式商业版以 **GEO（Generative Engine Optimization / 生成式引擎优化 / AI 搜索优化）** 作为第一切入口，帮助企业提升以下资产在 **AI 搜索与大模型回答场景** 中的可见度、引用率、推荐率与可信度：

- 品牌资产
- 官网与产品页
- 内容（问答、百科、媒体稿、行业文章、产品介绍）
- 证书 / 资质 / 案例素材
- 第三方背书（媒体稿、百科、问答口碑）

并通过 **AI 超级员工** 形成从 **AI 搜索曝光 → 线索跟进 → 内容生产 → 合规审查 → 效果复盘** 的业务闭环。

MOY 不是工具集合，而是 **操作系统**：统一调度数据、智能体、任务流与权限，把 AI 搜索可见度变成可度量、可执行、可交付的企业增长动作。

## 2. MOY 1.0 的边界

### 2.1 1.0 必须交付（In Scope）

1. **GEO 工作台**（完整闭环）
   - 品牌资产管理、客户企业资料管理、品牌知识库、产品/服务知识库、证书/资质/案例素材库
   - 关键词与提示词矩阵（品牌 × 人群 × 场景 × AI 平台）
   - AI 搜索平台监测（ChatGPT、Perplexity、Google AI Overviews、Claude、Gemini、Kimi、豆包、DeepSeek、文心、智谱等）
   - 品牌提及率、引用率、推荐率、竞品对比、负面风险监测
   - GEO 任务计划、内容生产任务、内容发布/交付管理、效果复盘报告
2. **AI 超级员工**（6 类角色）
   - AI 内容运营、AI 销售、AI 客服、AI 美工、AI 法务/合规、AI 数据分析
3. **企业级后台**
   - 多租户企业管理、用户/角色/权限、操作审计日志、API Key 管理、文件上传与资产管理、系统设置、任务队列、通知中心、数据看板
4. **UI/UX**
   - Web 端单端交付，深色未来科技感、克制、专业
   - 核心体验：指挥中心、任务流、智能体协同、增长雷达、品牌可见度地图

### 2.2 1.0 明确不做（Out of Scope）

- 移动端 App（1.0 只做 Web，移动端响应式适配可做，但不做原生 App）
- 通用 RAG 知识库问答产品（GEO 不是 RAG，见 §4）
- 通用企业 AI 平台 / 低代码 Agent 编排平台（这是 2.0/3.0 的事）
- 公域社交内容自动分发（1.0 只做内容生产与交付管理，不做自动发布到第三方平台）
- 自研大模型（MOY 永远不做基础模型，只做应用层编排）
- 多语言国际化（1.0 中文优先，i18n 架构预留但不在 1.0 落地多语言）

## 3. 为什么不做 MVP

| MVP 思维 | MOY 1.0 思维 |
|---|---|
| 用最少代码验证假设 | 用完整产品交付商业价值 |
| 半成品可上线 | 半成品不可交付客户 |
| 功能可以临时堆 | 功能必须有清晰边界、权限、审计、测试 |
| 前端假页面可以接受 | 前端假页面禁止，除非明确标注 UI 原型阶段 |
| 错误处理可以后补 | 错误处理必须随功能一起交付 |
| 不考虑多租户隔离 | 多租户隔离是第一公民 |
| 不考虑审计 | 审计日志必须随关键操作一起落地 |

**MOY 1.0 = First Complete Product**：第一个正式商业版本，面向真实付费客户交付。任何"先这样后面再补"的代码都不允许进入主干。

## 4. GEO 的正确定义

### 4.1 GEO 是什么

GEO（Generative Engine Optimization）是针对 **生成式 AI 搜索引擎与大模型对话场景** 的品牌可见度优化方法论。

其优化对象是：当用户在 ChatGPT、Perplexity、Google AI Overviews、Kimi、豆包、DeepSeek 等场景中提出与品牌、产品、行业相关的问题时，**AI 是否提及、引用、推荐本品牌，以及以何种可信度呈现**。

### 4.2 GEO 不是什么（关键边界，禁止混淆）

| 容易混淆的概念 | 与 GEO 的区别 |
|---|---|
| RAG（检索增强生成） | RAG 是给 LLM 喂私有知识库的技术手段；GEO 是优化品牌在第三方 AI 平台回答中的可见度，不依赖私有知识库注入 |
| 企业知识库问答 | 知识库问答服务企业内部；GEO 服务品牌在公域 AI 平台的可见度 |
| 传统 SEO | SEO 优化搜索引擎结果页排名；GEO 优化 AI 生成回答中的提及与引用，算法、指标、方法完全不同 |
| 舆情监控 | 舆情监控只读不优化；GEO 包含监测 + 内容生产 + 持续优化闭环 |

### 4.3 GEO 的核心指标（1.0 必须度量）

1. **可见度（Visibility）**：在目标 AI 平台、目标提示词下，品牌是否出现在回答中
2. **提及率（Mention Rate）**：被提及的提示词占比 = 被提及提示词数 / 总提示词数
3. **引用率（Citation Rate）**：回答中引用品牌官网/权威来源的占比
4. **推荐率（Recommendation Rate）**：品牌被 AI 主动推荐为解决方案的占比
5. **可信度（Trust Score）**：AI 回答中品牌呈现的正向/中性/负面倾向
6. **获客转化（Lead Conversion）**：从 AI 搜索曝光到线索进入销售漏斗的转化（1.0 度量线索，不度量最终成交）

### 4.4 GEO 的方法边界声明（合规与诚实）

- MOY 通过 **真实 AI 平台 API 测试 + 人工复核** 度量上述指标
- MOY **不保证** 排名提升、不保证 AI 一定推荐、不保证获客转化
- MOY 报告必须包含 **方法边界声明**，明确测试样本、测试时间、API 局限、非保证条款
- 真实测试结果默认 PENDING 状态，必须经人工复核才能标记 VERIFIED
- Mock 数据必须与真实数据严格区分（batch label 含 mock/demo 标记），不得伪装成真实客户数据

## 5. AI 超级员工与 GEO 的关系

AI 超级员工不是独立的"AI 助手玩具"，而是 **GEO 业务闭环的执行单元**：

```
GEO 工作台（发现问题、定义任务）
    ↓
AI 内容运营 Agent（生产 GEO 内容：问答、百科、媒体稿、行业文章）
    ↓
AI 法务/合规 Agent（审查违规、夸大、敏感风险）
    ↓
AI 美工 Agent（生成配图、海报思路、视觉素材需求）
    ↓
内容发布/交付管理
    ↓
AI 搜索平台监测（度量可见度、引用率、推荐率变化）
    ↓
AI 数据分析 Agent（生成周报、月报、客户交付报告、优化建议）
    ↓
AI 销售 Agent（基于 GEO 线索分级、跟进建议、销售话术、邮件草稿）
    ↓
AI 客服 Agent（沉淀客户真实问答，反哺 GEO 问答资产库）
    ↓
回到 GEO 工作台（效果复盘 → 新一轮任务计划）
```

**关键原则**：
- 每个 Agent 有明确职责边界，不是通用聊天机器人
- 每个 Agent 输出必须可被人工审核拦截（human-in-the-loop）
- 每个 Agent 有防幻觉机制（引用证据、置信度、最大 LLM 调用次数）
- 每个 Agent 任务有完整执行日志，可审计、可复盘

## 6. 首批目标客户

### 6.1 理想客户画像（ICP）

- **行业**：B2B 科技/SaaS、专业服务（律所/咨询/财税）、高端制造、医疗健康、教育科技
- **规模**：50-2000 人，年营收 5000 万-20 亿
- **特征**：
  - 已有品牌官网与一定内容资产
  - 决策者已意识到"AI 搜索正在改变获客方式"
  - 有市场团队但缺乏 AI 搜索优化能力
  - 愿意为"可度量的 AI 可见度提升"付费
  - 对合规与数据安全有基本要求

### 6.2 不服务（1.0 阶段）

- 个人创作者、个体户
- 灰产、黑产、敏感行业（赌博、色情、医疗违规、金融违规）
- 需要自动发布到第三方平台、自动刷量的客户
- 期望"保证排名第一""保证 AI 一定推荐"的客户

## 7. 产品不可妥协原则

1. **不做假数据**：Mock 数据必须标记，真实数据必须人工复核，报告必须含方法边界声明
2. **不做假页面**：前端页面必须有真实 API 支撑，禁止 mock 假接口冒充真接口
3. **不绕过多租户隔离**：任何 service 层查询必须带 tenantId，禁止跨租户访问
4. **不绕过审计**：关键操作（登录、权限变更、内容交付、报告导出）必须写审计日志
5. **不裸抛错误**：所有业务错误使用 AppError 体系，禁止 throw 字符串或裸 Error
6. **不跳过测试**：核心逻辑必须有单测，API 必须有集成测试，关键流程必须有 e2e
7. **不引入无必要复杂度**：但必须为多租户、权限、审计、Agent 调度预留架构空间
8. **不把 GEO 写成 RAG**：GEO 核心是 AI 搜索可见度，不是私有知识库问答
9. **不把 MOY 写成 MVP**：1.0 是 First Complete Product，半成品不交付
10. **不破坏既有约束**：本宪章与项目记忆中的硬约束冲突时，提请产品负责人裁决，不擅自放宽
