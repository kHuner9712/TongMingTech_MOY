## 阶段 0：工程底座与项目宪法（已完成）

### 交付内容
- Monorepo（pnpm workspace）+ apps/web + apps/api + packages/shared + packages/ui + packages/config
- TypeScript 严格模式 + ESLint + Prettier + Vitest + Docker Compose + GitHub Actions CI
- 统一错误体系（AppError + ErrorCode）+ Result 类型 + 结构化日志（pino + 脱敏）
- Fastify API 壳：健康检查（live/ready）+ 请求上下文 + 错误处理 + Prisma/Redis 插件
- Prisma schema 基线：Tenant / User / UserRole / RefreshToken / AuditLog
- Zod 环境变量校验（fail-fast）
- Next.js Web 壳：深色未来科技感首页
- 6 份项目宪法文档：PRODUCT_CHARTER / ARCHITECTURE / DEVELOPMENT_STANDARDS / DATABASE_PLAN / AGENT_ARCHITECTURE / ROADMAP

### 交付标准（验收）
- `pnpm install` 成功
- `pnpm lint` `pnpm typecheck` `pnpm test:unit` `pnpm build` 全绿
- `pnpm prisma:validate` 通过
- API 启动后 `/health` `/v1/health/live` `/v1/health/ready` 返回正确
- Web 首页渲染工程底座状态页

### 明确不做
- 任何业务功能（GEO / Agent / 后台 CRUD）
- 鉴权实现（只留接口与 schema）
- 真实 LLM 调用

## 阶段 1：企业级后台基线

### 功能边界
1. 鉴权：注册（邀请制）/ 登录 / 登出 / 刷新令牌 / 密码重置
2. 多租户：租户 CRUD（平台管理员）+ 租户切换
3. 用户与权限：用户 CRUD + 角色分配 + 权限校验中间件
4. 审计日志：审计日志查询 + 导出（按租户/用户/动作/时间）
5. API Key 管理：创建/撤销/权限范围
6. 文件资产：上传/下载/签名 URL（local + s3 驱动）
7. 通知中心：站内信 + 邮件（占位，邮件阶段 4 接入）
8. 系统设置：租户级配置
9. Web 端：登录页 + 工作台框架（侧边栏 + 顶栏 + 路由守卫）+ 上述功能页面

### 交付标准
- 所有 API 有 Zod schema + 集成测试（happy + error path）
- 多租户隔离测试通过（A 租户无法访问 B 租户数据）
- 审计日志覆盖所有关键操作
- e2e：登录 → 创建子用户 → 分配角色 → 该用户登录访问受限资源被拒
- 覆盖率：api 模块 ≥ 75%

## 阶段 2：GEO 工作台核心闭环

### 功能边界
1. 客户企业资料管理
2. 品牌资产管理（品牌/产品/服务/证书/案例）
3. 品牌知识库 + 产品知识库
4. 关键词与提示词矩阵（品牌 × 人群 × 场景 × AI 平台）
5. AI 搜索平台监测：批量测试任务（接入 ChatGPT/Perplexity/Kimi/豆包/DeepSeek 等真实 API）
6. 测试结果管理：PENDING → VERIFIED / REJECTED 人工复核流
7. 核心指标计算：提及率/引用率/推荐率/可信度（仅基于 VERIFIED 结果）
8. 竞品对比 + 负面风险监测
9. GEO 任务计划 + 内容生产任务 + 内容发布/交付管理
10. 效果复盘报告（Markdown + HTML，含方法边界声明）
11. 报告交付状态机：DRAFT → READY_FOR_CLIENT → DELIVERED → ARCHIVED
12. Web 端：指挥中心、提示词矩阵、批次运行、结果复核、报告中心

### 交付标准
- 真实 AI 平台 API 测试通过（至少 3 个平台）
- Mock 数据严格标记，真实数据默认 PENDING
- 报告核心指标仅来自 VERIFIED 结果，无 VERIFIED 时禁止生成报告
- 报告含方法边界声明
- 交付就绪检查脚本通过
- e2e：创建项目 → 配置矩阵 → 批次运行 → 复核结果 → 生成报告 → 交付

## 阶段 3：AI 超级员工 6 类角色

### 功能边界
1. LlmProvider 抽象层 + OpenAI/Anthropic/国产模型接入
2. PromptTemplate 管理 + 版本化 + 租户派生
3. 6 类 Agent 实现：
   - 内容运营（与 GEO 内容生产任务联动）
   - 销售（线索分级 + 话术 + 邮件草稿）
   - 客服（问答沉淀反哺资产库）
   - 美工（视觉需求说明）
   - 法务/合规（强制阻断型审查）
   - 数据分析（报告生成辅助）
4. Agent 任务队列（BullMQ）
5. 人工审核 UI（审核任务列表 + approve/reject/revision）
6. Agent 执行日志可视化
7. 防幻觉机制全量落地（引用证据 + 置信度 + maxLlmCalls）
8. Web 端：智能体协同工作区、审核中心、任务流

### 交付标准
- 每个 Agent 有单测 + 集成测试
- 防幻觉机制测试通过（低置信度强制审核、超 LLM 调用次数终止）
- 人工审核流 e2e 通过
- Agent 输出含引用证据，可溯源
- token 计量准确

## 阶段 4：1.0 正式商业版交付

### 功能边界
1. 阶段 1-3 全部功能稳定
2. 计费与配额（租户级 LLM 调用配额 + 任务配额）
3. 邮件通知接入（SMTP / 第三方邮件服务）
4. 数据看板（指挥中心聚合视图）
5. 客户交付报告模板库
6. 安全加固：速率限制、CSRF、安全头、依赖扫描
7. 性能优化：缓存、N+1 消除、慢查询治理
8. 部署：生产环境 Docker 镜像 + CI/CD + 数据库迁移流程
9. 文档：用户手册、管理员手册、API 文档（OpenAPI）
10. 售前 demo 环境与样例数据（严格标记 mock）

### 交付标准
- 安全审计通过（OWASP Top 10 覆盖）
- 压测：100 并发 API P95 < 500ms
- 至少 1 家设计伙伴客户真实使用 ≥ 4 周
- 客户交付报告通过交付就绪检查
- 全部 CI 流程绿
- 1.0 发布候选版本（RC）冻结

### 1.0 边界重申
- 不做移动 App
- 不做公域自动分发
- 不做通用 RAG 问答产品
- 不做多语言（架构预留）
- 不做自研模型

## 长期愿景：Mate Of You

MOY 的长期愿景是成为企业的 **AI 增长伙伴**（Mate Of You）：

- 不只是工具，而是与企业共同成长的 AI 员工团队
- 不只做 GEO，而是覆盖企业所有 AI 增长场景
- 不只提供服务，而是与企业共建 AI 增长能力
- 最终成为企业在 AI 时代的 **增长操作系统**，让每一家企业都能像有顶级 AI 团队一样增长

### 不变的原则（贯穿所有版本）
1. 不做假数据
2. 不绕过多租户隔离与审计
3. 不把 MOY 写成 MVP
4. 不把 GEO 写成 RAG
5. 不自研基础模型，永远做应用层
6. 不为短期交付牺牲可维护性

---

## 阶段交付节奏原则

- 每阶段必须有明确的功能边界与交付标准，不蔓延
- 每阶段交付前必须通过全部 CI + 交付就绪检查
- 跨阶段的功能依赖必须显式声明，不隐式耦合
- 阶段内可调整细节，但不可跳过交付标准
- 阶段交付后冻结 1 周，期间只修 bug，不加功能
